# AI-GAP 자동매매 프로그램 요구사항 문서 v1.0

## 1. 프로젝트 개요

프로젝트명은 `AI-GAP`이다.

본 프로그램은 한국 주식시장에서 장 시작 직후 갭상승 종목을 탐지하고, 그중 ETF/ETN 등 상품형 종목을 제외한 뒤, 거래대금·갭상승률·시초가 유지 여부·고가 재돌파 가능성 등을 기준으로 상위 15개 종목을 선정하여 예산에 맞게 자동 매수하고, 당일 중 목표수익 또는 손절 조건에 따라 자동 매도하는 프로그램이다.

본 프로그램은 실전 자동매매를 목표로 하되, 최초 버전에서는 반드시 모의투자 또는 드라이런 모드를 기본값으로 구현한다.

## 2. 작업 폴더

다음 폴더를 기준으로 프로젝트를 생성한다.

`%USERPROFILE%\Desktop\AI-GAP`

이미 폴더가 존재하면 기존 파일을 분석한 후 필요한 파일만 추가·수정한다. 기존 코드가 있다면 임의 삭제하지 말고 백업 후 수정한다.

## 3. 핵심 매매 전략

### 3.1 전략명

장초반 갭상승 강세주 선별 매매 전략

### 3.2 기본 아이디어

장 시작 직후 갭상승한 종목 중에서 실제 수급이 강하고 추가 상승 가능성이 높은 종목만 선별한다. 단순히 갭상승률이 높은 종목을 무조건 매수하지 않는다.

### 3.3 매매 흐름

1. 09:00 장 시작 후 갭상승 종목 목록을 수집한다.
2. 갭상승 상위 종목 중 ETF, ETN, 우선주, 스팩, 관리종목, 거래정지 종목, 투자주의/경고/위험 종목은 제외한다.
3. 거래대금, 갭상승률, 시가 대비 현재가 유지 여부, 당일 고가 재돌파 여부, 전일 거래대금 대비 당일 초반 거래대금 강도 등을 기준으로 점수를 계산한다.
4. 점수가 높은 상위 15개 종목을 선정한다.
5. 사용자가 설정한 총 예산을 상위 15개 종목에 배분한다.
6. 각 종목별 주문 가능 수량을 계산하여 시장가 또는 지정가로 매수한다.
7. 매수 후 실시간으로 수익률을 추적한다.
8. +3% 수익 도달 시 해당 종목 보유수량의 50%를 매도한다.
9. +5% 수익 도달 시 해당 종목의 나머지 잔여수량을 전량 매도한다.
10. 손절 조건 또는 시간 청산 조건에 해당하면 전량 매도한다.
11. 장 종료 전에는 모든 포지션을 정리하여 오버나이트 보유를 하지 않는다.

## 4. 데이터 수집 소스

### 4.1 1순위 데이터 소스

한국투자증권 Open API를 1순위 데이터 소스로 사용한다.

필요 데이터는 다음과 같다.

* 국내주식 현재가
* 시가
* 고가
* 저가
* 현재가
* 전일 종가
* 등락률
* 거래량
* 거래대금
* 실시간 체결가
* 호가
* 주문 가능 금액
* 보유잔고
* 주문 체결 내역

### 4.2 2순위 보조 데이터 소스

네이버증권 홈페이지를 보조 데이터 소스로 사용할 수 있다.

단, 네이버증권은 공식 Open API가 아니므로 HTML 구조 변경 가능성을 고려한다. 네이버증권 크롤링은 보조 수단으로만 사용하고, 실전 주문 판단은 가능한 한 한국투자증권 API 데이터를 우선한다.

### 4.3 갭상승 종목 수집 방식

다음 두 가지 방식을 모두 지원하도록 설계한다.

#### 방식 A: 한국투자증권 API 기반

한국투자증권 API에서 제공하는 순위, 조건검색, 현재가, 실시간 시세 관련 API를 조합하여 갭상승률이 높은 종목을 수집한다.

#### 방식 B: 네이버증권 갭상승 페이지 기반

네이버증권 갭상승 페이지에서 종목명, 종목코드, 현재가, 시가, 고가, 저가, 등락률, 거래량, 거래대금 정보를 수집한다.

네이버 페이지 구조가 변경되거나 수집 실패 시 프로그램은 중단하지 말고 한국투자증권 API 방식으로 대체한다.

## 6. 매수 후보 조건

### 6.1 기본 조건

다음 조건을 만족하는 종목만 매수 후보로 둔다.

* 갭상승률: +3% 이상 +15% 이하
* 09:03 이후 현재가가 시가 대비 -1.5% 이상 유지
* 09:05 기준 거래대금 30억 원 이상
* 전일 대비 거래량 또는 거래대금 증가
* 당일 저가가 시가 대비 과도하게 이탈하지 않을 것
* 당일 고가 재돌파 또는 고가 부근 유지
* 호가 스프레드가 과도하지 않을 것
* 주문 가능 수량이 충분할 것

### 6.2 갭상승률 계산

갭상승률은 다음 기준으로 계산한다.

`갭상승률 = (당일 시가 - 전일 종가) / 전일 종가 * 100`

### 6.3 시초가 유지율

시초가 유지율은 다음 기준으로 계산한다.

`시초가 유지율 = (현재가 - 시가) / 시가 * 100`

시초가 유지율이 -1.5% 미만이면 매수 후보에서 제외한다.

## 7. 종목 점수화 로직

각 후보 종목에 대해 종합 점수를 계산한다.

총점은 100점 기준으로 한다.

### 7.1 점수 구성

1. 갭상승률 점수: 20점
2. 거래대금 점수: 25점
3. 시초가 유지 점수: 20점
4. 고가 재돌파/고가 근접 점수: 15점
5. 전일 거래대금 대비 당일 거래대금 강도: 10점
6. 변동성 안정성 점수: 10점

### 7.2 감점 요소

다음 조건에 해당하면 감점한다.

* 갭상승률이 15% 초과: 감점
* 시초가 이후 급락 후 반등한 종목: 감점
* 거래대금은 크지만 윗꼬리가 긴 종목: 감점
* 호가 스프레드가 큰 종목: 감점
* 최근 3일 급등률이 과도한 종목: 감점
* 뉴스 없이 비정상 급등한 종목: 감점
* 우선주·ETF·ETN 의심 종목: 즉시 제외 또는 강한 감점

## 8. 매수 종목 수

최종 매수 종목 수는 최대 15개로 한다.

후보 종목이 15개 미만이면 조건을 만족하는 종목 수만 매수한다.

조건을 만족하는 종목이 없으면 매수하지 않는다.

## 9. 예산 배분

### 9.1 기본 예산 배분

사용자가 설정한 총 예산을 최종 선정된 종목 수로 균등 배분한다.

예시:

* 총 예산: 3,000,000원
* 최종 선정 종목: 15개
* 종목당 배정 예산: 200,000원

### 9.2 주문 수량 계산

각 종목별 주문 수량은 다음과 같이 계산한다.

`주문수량 = floor(종목당 배정 예산 / 현재가)`

주문수량이 1주 미만이면 해당 종목은 매수하지 않는다.

### 9.3 고가 종목 처리

고가 종목으로 인해 1주 매수 시 종목당 배정 예산을 초과하는 경우, 다음 옵션 중 하나를 설정할 수 있게 한다.

* 옵션 A: 제외
* 옵션 B: 1주만 매수 허용
* 옵션 C: 예산 재배분

기본값은 옵션 A로 한다.

## 10. 매수 타이밍

09:00 정각 즉시 매수는 하지 않는다.

기본 매수 판단 시간은 다음과 같다.

* 1차 스캔: 09:03
* 2차 스캔: 09:05
* 최종 매수: 09:05~09:10

시가 이후 3~5분 동안의 가격 유지력과 거래대금을 확인한 후 매수한다.

## 11. 주문 방식

### 11.1 기본 주문 방식

초기 버전에서는 지정가 주문을 기본으로 한다.

지정가는 다음 중 하나를 선택할 수 있게 한다.

* 현재가
* 현재가 + 1틱
* 매도 1호가
* 사용자가 지정한 슬리피지 허용 범위 내 가격

기본값은 `매도 1호가` 또는 `현재가 + 1틱`으로 한다.

### 11.2 시장가 주문

시장가 주문은 옵션으로만 제공한다.

실전 모드에서는 사용자가 명시적으로 시장가 주문을 허용한 경우에만 시장가 주문을 사용한다.

## 12. 매도 전략

### 12.1 익절 조건

각 종목별 평균 매수가 대비 수익률을 기준으로 한다.

1. +3% 도달 시 보유수량의 50% 매도
2. +5% 도달 시 잔여수량 전량 매도

### 12.2 수량 처리

보유수량이 1주인 경우 +3% 도달 시 전량 매도하거나, 사용자가 선택할 수 있게 한다.

기본값은 다음과 같다.

* 보유수량 1주: +3% 도달 시 전량 매도
* 보유수량 2주 이상: +3% 도달 시 절반 매도, +5% 도달 시 잔여 전량 매도

### 12.3 손절 조건

기본 손절 조건은 다음과 같다.

* 평균 매수가 대비 -1.5% 손실 시 전량 매도
* VWAP 이탈 후 3분 이내 회복 실패 시 전량 매도
* 시가 이탈 후 회복 실패 시 전량 매도
* 10:30까지 수익 전환 실패 시 전량 매도 또는 후보별 판단
* 13:00까지 보유 중이면 전량 청산
* 장 종료 전 15:10까지 미청산 종목은 강제 청산

초기 버전에서는 VWAP 계산이 어려우면 평균 매수가 대비 -1.5% 손절과 시간 청산부터 구현한다.

## 13. 거래 시간 정책

프로그램은 한국 주식시장 정규장 기준으로 동작한다.

* 데이터 수집 시작: 08:55
* 갭상승 후보 수집: 09:00~09:05
* 매수 판단: 09:05~09:10
* 매수 실행: 09:05~09:10
* 모니터링: 매수 후 ~ 13:00
* 기본 청산: 13:00
* 비상 강제청산: 15:10

## 14. 모드 구분

프로그램은 다음 모드를 지원한다.

### 14.1 백테스트 모드

과거 데이터를 이용하여 전략 성과를 검증한다.

### 14.2 드라이런 모드

실시간 데이터를 사용하되 실제 주문은 넣지 않는다.

주문이 들어갔다고 가정하고 가상 포트폴리오를 기록한다.

### 14.3 모의투자 모드

한국투자증권 모의투자 API를 사용하여 주문을 실행한다.

### 14.4 실전투자 모드

실계좌 주문을 실행한다.

실전투자 모드는 기본 비활성화한다. 사용자가 설정 파일에서 명시적으로 활성화해야 한다.

## 15. 안전장치

다음 안전장치를 반드시 구현한다.

1. 실전투자 모드는 기본값 False
2. 1일 최대 투자금 제한
3. 종목당 최대 투자금 제한
4. 1일 최대 손실 제한
5. 주문 전 최종 확인 로그
6. 주문 실패 시 재시도 횟수 제한
7. API 오류 시 신규 매수 중단
8. 네트워크 오류 시 신규 매수 중단
9. 보유 종목 조회 실패 시 추가 매수 중단
10. 계좌 잔고 조회 실패 시 주문 중단
11. 장 종료 전 미청산 종목 강제 청산
12. 모든 주문과 체결 결과 로그 저장
13. 동일 종목 중복 매수 방지
14. 이미 매도한 종목 재매수 방지
15. 예외 발생 시 텔레그램 또는 콘솔 경고 출력

## 18. 주요 모듈 요구사항

### 18.1 kis_client.py

한국투자증권 API 연동을 담당한다.

기능:

* 접근토큰 발급
* 현재가 조회
* 시가/고가/저가/현재가 조회
* 거래량/거래대금 조회
* 잔고 조회
* 주문 가능 금액 조회
* 매수 주문
* 매도 주문
* 체결 내역 조회
* API 오류 처리
* 호출 제한 대응

### 18.2 naver_client.py

네이버증권 갭상승 페이지에서 보조 데이터를 수집한다.

기능:

* 갭상승 종목 리스트 수집
* 종목명/현재가/시가/고가/저가/등락률/거래량/거래대금 파싱
* HTML 구조 변경 또는 수집 실패 시 예외 처리
* 네이버 데이터는 보조 데이터로만 사용

### 18.3 gap_scanner.py

갭상승 후보 종목을 수집한다.

기능:

* 갭상승률 계산
* 후보 종목 정렬
* 상위 후보 추출
* 데이터 소스 장애 시 대체 소스 사용

### 18.4 filters.py

제외 대상 종목을 필터링한다.

기능:

* ETF 제외
* ETN 제외
* 우선주 제외
* 스팩 제외
* 리츠 제외
* 거래대금 부족 종목 제외
* 가격 조건 미달 종목 제외
* 호가 스프레드 과다 종목 제외

### 18.5 scoring.py

후보 종목의 점수를 계산한다.

기능:

* 갭상승률 점수
* 거래대금 점수
* 시초가 유지 점수
* 고가 근접 점수
* 전일 대비 거래대금 강도 점수
* 변동성 안정성 점수
* 감점 로직
* 최종 점수 산출

### 18.6 position_sizer.py

예산에 맞는 매수 수량을 계산한다.

기능:

* 총 예산 균등 배분
* 종목당 주문 가능 수량 계산
* 고가 종목 제외
* 남은 예산 재배분
* 최종 주문 리스트 생성

### 18.7 sell_rules.py

매도 조건을 판단한다.

기능:

* +3% 도달 시 절반 매도
* +5% 도달 시 잔여 전량 매도
* -1.5% 도달 시 손절
* 13:00 시간 청산
* 15:10 비상 청산
* 1주 보유 종목의 부분매도 예외 처리

### 18.8 order_manager.py

주문 실행과 상태 관리를 담당한다.

## 25. 중요 주의사항

본 프로그램은 투자 판단을 보조하는 자동화 도구이며 수익을 보장하지 않는다.

초기 개발 단계에서는 반드시 드라이런 또는 모의투자 모드로만 실행한다.

실전투자 모드는 충분한 백테스트와 모의투자 검증 후 사용자가 명시적으로 활성화한 경우에만 사용할 수 있도록 한다.

실전투자 모드에서도 1일 최대 투자금, 1일 최대 손실, 종목당 투자금 제한을 반드시 적용한다.
